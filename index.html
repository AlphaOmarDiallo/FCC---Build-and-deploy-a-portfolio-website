What I do

Native Android development

I build, manage, debug and publish Android applications on the Google play store. I use de Model View View Model architecture to build apps and the Unified Modeling Language logic to build databases.
I mainly use Kotlin as it is the official language for Android development. However, I am as comfortable using Java.
To keep improving my skills as a developer, I also build side projects to implement new features from Android.

Cross-platform development

I can build, manage, debug and publish apps on the Google play store and the Apple Appstore with Kotlin Multiplatform Mobile framework.
It does not impose any restrictions on how you develop your app’s UI or how you work with the platform, whenever a task cannot be solved in the shared code or whenever you want to use specific native features, you can use the expect/actual pattern to seamlessly write platform-specific code.
